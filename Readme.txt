Written in VS2012
Requires: OpenXMLSDK2.5
Uses Entity Framework for DB connection

To run; delete packages directory and packages.config, get (NuGet) entity framework, install OpenXMLSDK2.5, ensure the path 
"C:\GeneratedSpreadsheets\" exists or change it in Program.cs.

The generated file will be named "out.xlsx" and will open in the Excel version that comes with office 365
